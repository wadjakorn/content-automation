# Lane D — Context/Timing risk

ไม่ดูตัวเนื้อหา แต่ดูบริบทรอบการเผยแพร่ — รันที่ pre-publish (Gate 3) เป็นหลัก

## D1 — Disclosure (compliance, บางส่วน hard)
detect ความสัมพันธ์เชิงพาณิชย์ (sponsored/ของแถม-PR/affiliate/เครื่องที่แบรนด์ส่ง) → ต้อง disclose
- YouTube: ติ๊ก paid promotion (ระบบโชว์ disclosure อัตโนมัติ) + พูด/เขียน desc; ครีเอเตอร์รับผิดชอบกฎหมายท้องถิ่น; ครอบคลุมแม้ของฟรี
- ไทย: พ.ร.บ.คุ้มครองผู้บริโภค ม.22 ห้ามโฆษณาเท็จ/เกินจริง/ทำให้เข้าใจผิด (สคบ. บังคับใช้)
- **healthcare pillar = ทุ่นระเบิด:** อย. ถือว่ารีวิวเพื่อการค้า = โฆษณา; เคลมสรรพคุณเกินจริง โทษจำคุก ≤3 ปี/ปรับ ≤30,000 บาท; ต้องมีเลข ฆอ.; อย. สั่งแพลตฟอร์มปิดกั้นได้ → **เคลมสุขภาพ = hold เสมอ**
- hard: รู้ว่า sponsored แต่ disclosure หาย → block จนกว่าจะ disclose

## D2 — Sensitivity / timing (soft แต่ hold ได้)
sensitivity calendar: อย่าปล่อย hype/มุก/ขายของช่วงไว้ทุกข์ระดับชาติ/ภัยพิบัติ/โศกนาฏกรรม
(ไทยอ่อนไหวเป็นพิเศษช่วงไว้ทุกข์ที่เกี่ยวกับสถาบัน); เช็คเวลา publish กับปฏิทิน → ขัด = เสนอเลื่อน

## D3 — Consistency (advisory)
RAG เหนือ transcript เก่า flag ความขัดแย้งกับจุดยืนที่เคยพูด → ให้ acknowledge;
+ consistency ข้ามแพลตฟอร์ม (ราคา/เคลมตรงกัน)

## Synthesis — "backlash prevent"
rollup B (tone) + C (accuracy) + D (context) → `reputational_risk` เดียว ใช้ที่ Gate 3
Schema: `schemas/lane-d-context.schema.json`

## Reality check
disclosure (โดยเฉพาะ health) เป็นกฎหมายจริง → healthcare default hold; sensitivity = ประกันราคาถูก; consistency = nice-to-have
